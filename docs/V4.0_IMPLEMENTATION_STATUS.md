### 补完任务 1：创建 markdown_chunker.py（对应 A2）

**文件**：`backend/app/services/markdown_chunker.py`（新建）

**要求**：
- 导出函数 `split(markdown: str, source_url: str, library: str, version: str) -> list[DocumentChunk]`
- 第一阶段：用 `langchain_text_splitters.MarkdownHeaderTextSplitter` 按 h1/h2/h3 切出语义单元，保留标题路径（如 "Guide > Reactivity > ref()"）
- 第二阶段：对每个语义单元做代码感知递归切分——把 ` ``` ` 围栏块视为原子单元不拆分；超长代码块（超过 chunk_size）单独成 chunk 并从围栏标注提取语言（如 python/vue/typescript）
- 普通文本部分仍用 RecursiveCharacterTextSplitter，分隔符增加代码块边界
- 产出的 DocumentChunk 填充 source_url/library/version/heading_path/code_language/is_code_block
- chunk_id 用确定性哈希（见补完任务 5）

**验收**：
- `markdown_chunker.split("...\n```python\nprint('hi')\n```\n...", "https://x.com", "vue", "3")` 返回的 chunk 中代码块完整、is_code_block=True、code_language="python"

### 补完任务 5：确定性 chunk_id（对应 B3）

**文件**：`backend/app/services/document_service.py`、`backend/app/services/mineru_document_service.py`

**当前状态**：`document_service.py` 第 82 行 `chunk_id=uuid4().hex`；`mineru_document_service.py` 第 360 行 `_make_chunk` 同样 `uuid4().hex`。

**改造内容**：
```python
import hashlib

def _deterministic_chunk_id(library: str, source_url: str, heading_path: str, content: str) -> str:
    raw = f"{library}|{source_url}|{heading_path}|{content[:200]}"
    return hashlib.sha256(raw.encode()).hexdigest()[:32]
```
- PDF 摄取路径：library 用 document_name 或 "default"，source_url 用 file_path
- URL 摄取路径：library/source_url/heading_path 来自抓取元数据

**验收**：同一文档两次入库，chunk 数量不变（upsert 覆盖而非新增）
